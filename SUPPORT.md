# Support

## Community Support

Use GitHub Issues and Discussions for:

- installation problems;
- privacy-safe reproduction cases;
- scene-plan and configuration questions;
- provider-adapter proposals;
- documentation corrections;
- platform preset and QA-rule suggestions.

Community support is best-effort. There is no guaranteed response time.

## What Not To Post Publicly

Do not paste or upload:

- API keys, Authorization headers, cookies, signed URLs, or `.env` files;
- private portraits, voice samples, client videos, or brand assets;
- real provider job state;
- paid-account screenshots with account IDs or billing data;
- private local paths that reveal a client or personal identity.

Use GitHub private vulnerability reporting for security issues.

## Commercial Support Boundary

The open-source project is a self-serve orchestration Skill. It does not include:

- provider accounts or credits;
- installation on your machine;
- customization for your article archive, brand, avatar, voice, or publishing workflow;
- team SOPs, review processes, or ongoing maintenance;
- guaranteed output quality for private production assets.

Commercial services, if offered separately, should be framed around implementation, customization, training, and production reliability, not around selling access to this public repository.
