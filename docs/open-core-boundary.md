# Open Core Boundary

This repository is designed to earn trust without giving away the whole production system.

## Open source in this repository

The public project may include:

- the Codex skill instructions;
- article-to-video workflow gates;
- portable project contracts;
- sample article, content brief, voiceover, pronunciations, scene plan, and QA report;
- validation scripts;
- local/mock examples;
- provider adapter contracts and documentation;
- basic cover, caption, effect, and export ideas as examples;
- safety, consent, cost, and release-readiness checks.

These pieces help users understand the method and verify that the workflow is real.

## Not included here

The public project should not include:

- paid Obsidian workbench source code;
- historical project management UI;
- commercial project templates;
- one-click cover image generation workflows;
- Jianying/CapCut draft-package generator implementation;
- private client projects;
- customer delivery SOPs;
- high-converting sales scripts;
- private avatar, portrait, voice, or brand assets;
- provider accounts, credits, keys, cookies, sessions, signed URLs, or local absolute paths.

These are product and service assets, not open examples.

## Product ladder

```text
Open source
  Skill + protocol + examples + validation

Paid course / kit
  Obsidian workbench + project pack + templates + guided workflow

High-ticket service
  Custom video production system + content library integration + delivery SOP
```

The open-source layer sells possibility and trust. The paid layer sells speed, certainty, templates, and implementation.

## Practical rule

Open what lets a developer or creator say:

```text
"I understand how this system thinks."
```

Do not open what lets them say:

```text
"I can now run the full commercial production workflow without the paid product."
```

That is the boundary.
