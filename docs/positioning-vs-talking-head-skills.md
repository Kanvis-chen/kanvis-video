# Positioning Against Talking-Head Skills

This document is for maintainers. It keeps the public story clear without naming or attacking any neighboring project.

## Public Positioning

Use this sentence:

```text
Kanvis Cut turns long-form content into directed, release-gated video projects. Its first workflow, `article-to-avatar-video`, supports avatar presenter output without making the avatar the center of the product.
```

Do not use:

```text
alternative to <competitor>
digital-human production clone
HeyGen + MiniMax workflow
one-click talking-head generator
```

## Why This Is A Different Project

| Question | Talking-head production skill | Kanvis Cut |
|---|---|---|
| What is the input? | Finished script, portrait, voice sample | Article, WeChat post, Markdown, course draft |
| What is the hard problem? | Authorized provider calls and asset readiness | Editorial adaptation, semantic direction, visual evidence |
| What does the avatar do? | Usually the main visual | Presenter, guide, PIP, or absent when the content needs space |
| What is the core artifact? | Provider job state and final talking-head video | Content brief, voiceover, scene plan, graphics, QA report |
| What should contributors improve? | Provider integrations and generation state | Scene grammar, article fidelity, visual roles, platform QA |

The overlap is normal: consent, cost control, preview approval, and provider adapters are table stakes for any responsible avatar workflow. They are not the project identity.

## Public Demo Rule

Every public demo should show the article-to-video difference:

```text
same article paragraph
-> flat avatar reading
-> directed video scene with evidence / process / comparison / PIP
```

If the demo only shows a face speaking, it weakens the project positioning.

## Contribution Filter

Accept contributions that strengthen:

- source-faithful article adaptation;
- Chinese pronunciation and terminology;
- semantic scene grammar;
- information graphics and presenter PIP layouts;
- platform presets and safe margins;
- quality gates and privacy-safe examples;
- provider adapters that remain replaceable.

Push back on contributions that make the project a generic:

- avatar provider wrapper;
- voice cloning wrapper;
- full-screen digital-human generator;
- scraping or publishing automation tool;
- hosted SaaS backend.

## Short Answer For Comparisons

If someone asks whether this is inspired by or copying a digital-human talking-head repo, answer:

```text
No. This project sits one layer earlier in the workflow. It starts from long-form Chinese content and focuses on editorial adaptation, semantic scene planning, information graphics, presenter PIP, and release QA. Provider calls are replaceable edges, not the center of the project.
```
